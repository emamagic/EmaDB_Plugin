Id Annotation Inspection

Description

The Id Annotation Inspection ensures that fields annotated with @Id meet required criteria, providing reliable validation for primary key fields in your entity classes.

Validation Rules

This inspection checks the following:

  • Only one field per class may be annotated with @Id.
  • For PostgreSQL databases, the @Id field type must be either Integer or String.
  • For MongoDB databases, the @Id field type must be org.bson.types.ObjectId.

Usage

This inspection helps ensure that primary key fields in your entity classes conform to database-specific requirements, promoting database integrity and data consistency.
